Architectural detail click to expand each layer

Layer 01 — Source, Input & Orchestrator Make.com

No API access to Lexis or Westlaw — confirmed by Seth. Sam is the human-in-the-loop. She logs in daily, downloads each complaint as a PDF, and drops the files into one watched Google Drive folder.

Orchestration platform: Make.com. A single Make scenario watches the Drive folder, pulls each new file, and runs the entire downstream chain — Claude parser, HubSpot dedupe, parallel enrichment HTTP requests, lead composition, HubSpot push, Gmail draft creation, Slack notify. No custom backend.

Module list: Google Drive (Watch Files in a Folder) → HTTP/Anthropic (Claude API) → HubSpot (search + create/update Contact, Company, Deal) → multiple HTTP modules for enrichment vendors → Gmail (Create Draft on rep's mailbox via OAuth) → Slack (Send DM to rep).

Layer 02 — AI Parser Agent Claude · Vision + LLM

A single Claude call per uploaded PDF that does:

  • File quality detection. State court PDFs are scanned/watermarked (Adame); federal e-filings are clean digital (Grant). OCR runs as needed — minor mangling like "Plaintift" still parses fine.
  • Vision pass. Detects and reads embedded images in the PDF — Grant's complaint embeds a phone screenshot; we extract sender number, sender domain, and message body from the pixels.
  • LLM extraction to a strict JSON schema (see Appendix A). One required pass + one self-check pass for accuracy.
  • Confidence scoring. Each field gets a confidence rating; low-confidence fields are flagged for Sam or rep review.
Layer 03 — HubSpot Dedupe three outcomes

Before spending money on enrichment APIs, hit HubSpot:

  • Existing customer match → notify their account manager, flag the customer record with a Litigation Filed tag, do not create a new lead.
  • Existing prospect/lead match → append the new case to the existing lead, notify the owning rep, do not double-assign.
  • No match → continue to enrichment.

Layer 05 — Plaintiff / Litigator Branch internal intelligence

A separate lane that compounds value over time:

  • Extract plaintiff's name, residence, counsel, and firm.
  • Cross-reference Blacklist's existing litigator database — is the plaintiff already known? Is the firm a serial filer (e.g., Jibrael Hindi)?
  • Track filing volume by firm, by week. The Adame + Grant pair (same firm, two states, two violation types, two days apart) is a real pattern that sharpens the sales conversation.

Decided 2026-05-07: Internal-use intelligence only in v1. Primary purpose is making the sales outreach hit harder ("we noticed Jibrael Hindi has filed three cases in the last two weeks against home-services companies — here's what they look for"). Productizing the report for clients is out of v1 scope; revisit once we have signal on data quality and demand.

Layer 06 — Lead Composition, Scoring, Assignment routing rules

The lead brief includes:

  • Defendant company profile (one-pager).
  • 1–3 decision-maker contacts with confidence ratings.
  • Case context: court, filing date, allegation type, damage exposure, plaintiff/counsel.
  • Tailored talking points by violation type — Federal DNC violation: lead with DNC scrubbing + litigator scrub. Internal DNC violation: lead with policies/procedures + opt-out automation. Express Consent: lead with consent capture + record retention.
  • Urgency score (filing date proximity, damage exposure, defendant size, contact confidence).

Routing: 5-rep round-robin across Kristyn, Dudley, Cierra, Maria, Suzi. Multi-defendant rule: when a complaint names multiple defendants, ALL route to the same rep in that turn — one owner pursues the whole case.

Layer 07 — HubSpot Push CRM write
  • Create Company record (with TCPA-specific custom properties — see Appendix B).
  • Create Contact records for each enriched decision-maker.
  • Create Deal/Lead with stage New TCPA Lead.
  • Attach AI-generated Lead Brief PDF + source complaint PDF.
  • Set lead owner = assigned rep.
  • Tag Source = TCPA Filed Case Lead Flow for clean conversion reporting.
Layer 08 — Outreach Email · drafted by AI, sent by rep human in the loop

AI drafts the outreach email using Blacklist Alliance brand voice and an allegation-type-specific template. Tone is respectful, direct, value-led — not ambulance chasing. Open with the case (factually, no editorializing), pivot to "here's what we do for TCPA defendants," CTA to a 15-min call.

v1 (locked): draft routes to a 1-click approval queue. Rep sees the draft, can edit inline, approves with one click and sends from their own mailbox.

v2 (future): auto-send with guardrails — min decision-maker confidence threshold, blacklist of competitor brands, mandatory human review every Nth send.

Why human-in-the-loop in v1: these are companies that just got sued. The first contact has to be excellent. One bad auto-send to the wrong person at a high-profile defendant becomes a screenshot in the wrong Slack channel. Plus: AI hallucinations (wrong damage estimate, fabricated officer name, wrong allegation type) get caught at this gate.

Layer 09 — Cadence and Reporting measure success

Auto follow-up Day 3, Day 7, Day 14 — same approval queue model in v1. Reply detection pauses cadence and notifies rep.

Daily digest to Esteban, Matt, and Seth: cases in / leads out / contacts found / sends / replies / meetings booked / closes.

Outside the automation — Jennifer builds a HubSpot deal flow & pipeline report separate from the daily digest. Standalone dashboard scoped to leads tagged Source = TCPA Filed Case Lead Flow so leadership can gauge whether the automation is actually working: leads created, stage distribution, contacts reached, meetings booked, deals won, win rate, time-to-first-touch, deal value.

Implications for the parser:

  • Must handle both state court scans (Adame is OCR'd, lower quality, watermarked) and clean federal e-filings (Grant). Modern LLMs handle this fine.
  • Must run vision extraction on embedded screenshots — Grant's complaint contains a phone screenshot showing the actual sender phone, sender domain, and message content. That's gold for the salesperson and feeds Number Verifier directly.
  • Must classify allegation type (Federal DNC vs. Internal DNC vs. Prior Express Consent vs. ATDS, etc.). The outreach pitch is materially different per type.
  • Must extract plaintiff counsel firm as a first-class field — that's where the litigator-firm intelligence comes from.
